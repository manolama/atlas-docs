    legend

    Input Stack:
    Query
    String
    		 ⇨ Output Stack:
    StyleExpression
     

    Set the legend text, often used to cleanup the default legend. Legends can contain variables based on the exact keys matched in the query clause and keys used in a group by. Variables start with a $ sign and can optionally be enclosed between parentheses. The parentheses are required for cases where the characters immediately following the name could be a part of the name. If a variable is not defined, then the name of the variable will be used as the substitution value.

    The variable atlas.offset can be used to indicate the time shift used for the underlying data and reduce the length of legend strings.

    Example including tag key:

    BeforeAfter
    name,sps,:eq,
(,nf.cluster,),:by

    name,sps,:eq,
(,nf.cluster,),:by,
Cluster:+$nf.cluster,:legend

    Example with custom title:

    BeforeAfter
    name,sps,:eq,
(,name,),:by

    name,sps,:eq,
(,name,),:by,
$(My+Custom+Text),:legend

    Example with an offset:

    BeforeAfter
    name,sps,:eq,
:sum,
1w,:offset

    name,sps,:eq,
:sum,
1w,:offset,
$(name)+$(atlas.offset),:legend
